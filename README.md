# 📚 Vocab Tester Project

**รายละเอียดโปรเจกต์สำหรับ AI Assistant (AI Context Document)**
เอกสารนี้มีไว้เพื่อให้ AI ตัวใหม่ (หรือแชทใหม่) เข้าใจภาพรวม โครงสร้าง และเป้าหมายของโปรแกรมนี้ได้ทันที โดยที่คุณไม่ต้องนั่งอธิบายใหม่ตั้งแต่ต้นครับ

---

## 📌 ภาพรวมโปรเจกต์ (Project Overview)
โปรแกรม **Vocab Tester** เป็นสื่อการสอนและเว็บแอปพลิเคชันแบบ **Offline 100%** ที่ออกแบบมาเพื่อการใช้งานในห้องเรียน (ฉายขึ้น TV หรือ Projector ด้วยสัดส่วน 16:9 แนวนอน) โดยมีฟีเจอร์หลักดังนี้:

1. **โหมดสอน (Teaching Mode):** พจนานุกรมภาพรวมคำศัพท์ทั้งหมดให้ครูเปิดสอนหน้าชั้นเรียน มีปุ่มกดฟังเสียงได้ทันที
2. **มินิเกมเพื่อการเรียนรู้:**
   - **เกมจับคู่ความจำ (Memory Match):** เปิดไพ่จับคู่รูปภาพกับคำศัพท์ พร้อมตัวจับจำนวนครั้งการเปิด (Moves)
   - **เกมสะกดคำ (Hangman):** ทายตัวอักษรเพื่อเติมคำศัพท์ให้สมบูรณ์ มีระบบหัวใจ (พลังชีวิต) 5 ดวง
   - **เกมแต่งประโยค (Sentence Mode):** ระบบ Fill in the blank ที่มีเทมเพลตประโยคสำเร็จรูปให้นักเรียนฝึกแต่งและพูดตอบ
3. **ระบบทดสอบนักเรียน:**
   - **Interactive Flashcard:** ทายคำศัพท์จากรูปหรือตัวหนังสือ (ดีไซน์แบบ Side-by-Side) พร้อมระบบประเมินเสียงพูด (Speech Recognition) ให้คะแนน 3 ส่วน (ความถูกต้อง, สำเนียง, ความลื่นไหล) โดยไมโครโฟนจะเปิดแช่ต่อเนื่องเพื่อความรวดเร็วในการสอน
   - **ระบบ QR Code:** มีปุ่มสร้าง QR Code ให้ผู้เรียนสแกนเพื่อเข้าใช้งานแอปผ่านอุปกรณ์ของตนเองได้ง่ายๆ
   - **โหมดกระดานจับคู่ (Worksheet):** แสดงรูปภาพพร้อมหมายเลข และคำศัพท์ด้านบน เพื่อให้นักเรียนทำแบบทดสอบลงในสมุด พร้อมฟีเจอร์สุ่มโจทย์
4. **Admin Panel & Offline Data:** มีหน้าจัดการหลังบ้าน (`admin.html`) ที่สามารถเพิ่ม/ลบหมวดหมู่ แล้ว Generate ออกมาเป็นไฟล์ `data.js` ได้เลย พกพาระบบใส่แฟลชไดรฟ์ไปใช้ที่ไหนก็ได้โดยไม่ต้องพึ่ง Database
5. **Smart Cropper V3:** เครื่องมือหั่นรูปภาพหนังสือขั้นสูง (`smart-cropper.html`) ที่ใช้ระบบ **Individual Draggable Boxes** (ลากกล่องอิสระทีละใบ ขยับ ย่อขยาย และลบทิ้งได้) พร้อมระบบ File System API ช่วยเซฟรูปลงโฟลเดอร์อัตโนมัติ

## 📂 โครงสร้างไฟล์ (File Structure)
โปรเจกต์นี้ทำงานด้วย HTML, CSS, Vanilla JS ล้วนๆ ไม่มี Framework

- `index.html` - หน้าจอหลักสำหรับนักเรียนเล่นเกมและทำแบบทดสอบ
- `style.css` - สไตล์และหน้าตาของแอปพลิเคชันหลัก (เน้น UI ทันสมัย Widescreen)
- `script.js` - โลจิกหลักของแอป เช่น ระบบสุ่มไพ่, โหมดเกมต่างๆ, ระบบ AI ตรวจจับเสียงพูด
- `data.js` - เก็บข้อมูลคำศัพท์ทั้งหมดในรูปแบบ Object ชื่อ `categoriesData` (ไฟล์นี้ได้มาจากการ Generate จากหน้า admin.html)
- `admin.html` - หน้าจอสำหรับคุณครู (Admin UI) ไว้เพิ่ม/ลบ/แก้ไข หมวดหมู่และคำศัพท์
- `smart-cropper.html` - เครื่องมือหั่นรูป V3 ช่วยคุณครูสร้างรูปภาพย่อยๆ จากหน้าหนังสือได้อย่างรวดเร็ว
- `images/` - โฟลเดอร์เก็บรูปภาพ โดยจะแบ่งซับโฟลเดอร์ตามรหัสหมวดหมู่ เช่น `images/vegetables/1.png`

## 🤖 คำแนะนำสำหรับ AI ในอนาคต (Instructions for Future AI)
หาก User เริ่มต้นการสนทนาใหม่แล้วแนบไฟล์นี้มาด้วย โปรดรับทราบหน้าที่ของคุณดังต่อไปนี้:

1. **หน้าที่ "ผู้ช่วยแยกคำศัพท์ (OCR Assistant)":** 
   - หาก User อัปโหลดรูปภาพหน้าหนังสือมาให้ **คุณห้ามใช้โค้ดดึงตัวอักษร** แต่ให้คุณใช้ Vision Model ของคุณ **อ่านภาพด้วยตัวเอง**
   - ดึงคำศัพท์ภาษาอังกฤษ, คำแปลภาษาไทย, และคำอ่านภาษาไทย ออกมาเป็น JSON Array
   - โครงสร้างที่ต้องทำคือ: `{ id: 1, en: "word", th: "คำแปล", pron: "คำอ่าน" }` 
   - ส่งข้อมูลนี้ให้ User เพื่อนำไปใส่ใน `admin.html` หรือนำไปเขียนทับใน `data.js`

2. **การทำงานออฟไลน์ (Offline Constraints):**
   - หากคุณต้องเขียนฟีเจอร์ใหม่ **ห้าม** เสนอเทคโนโลยีที่ต้องพึ่งพา Node.js Backend, Database (SQL/NoSQL) หรือ CDN ภายนอกเด็ดขาด!
   - ระบบทุกอย่างต้องทำงานแบบ Standalone ผ่านเบราว์เซอร์ `file:///` เท่านั้น (เพื่อการพกพาใน Flashdrive นำไปใช้ในโรงเรียนที่ไม่มีเน็ต)
   - ข้อมูลต่างๆ ต้องถูกบันทึกด้วยการประมวลผลเป็น Blob หรือใช้ File System Access API แล้วสั่งให้ Browser จัดการบันทึกทับไฟล์เดิม (เหมือนที่ทำใน `admin.html` และ `smart-cropper.html`)
   - **ระบบฟังเสียง (SpeechRecognition / TTS):** 
     - เนื่องจากบน `file:///` เบราว์เซอร์จะไม่จำสิทธิ์ไมโครโฟน จึงต้องบังคับตั้งค่า `continuous = true` ใน SpeechRecognition เสมอ เพื่อให้เบราว์เซอร์ถามสิทธิ์แค่ครั้งเดียวแล้วเปิดรับเสียงค้างไว้
     - **บั๊ก iOS Safari TTS:** การใช้งาน `window.speechSynthesis` บน iPad/iOS จะมีบั๊กที่ `getVoices()` คืนค่าเป็น Array ว่างในตอนแรก ทำให้ระบบไปดึงเสียง "ภาษาไทย" มาสะกดคำภาษาอังกฤษ (เช่น อ่าน carrot ว่า เคราท) วิธีแก้คือต้องดึง `getVoices()` ทิ้งไว้ล่วงหน้าผ่านอีเวนต์ `onvoiceschanged` และเก็บใส่ตัวแปร Global ไว้ใช้งาน พร้อมกับตั้งค่า `utterance.lang = 'en-US'` เสมอ
